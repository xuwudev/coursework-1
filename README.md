РОЗДІЛ 1
АНАЛІЗ ПРЕДМЕТНОЇ ОБЛАСТІ ТА ПРОБЛЕМАТИКА
 Об’єктно-орієнтоване програмування, як область застосування у розробці застосунку для ведення обліку візитів до лікаря.
1. Моделювання даних
ООП дозволяє створювати класи, які представляють об'єкти реальних систем. Наприклад, в контексті програмного забезпечення для управління курсами можна розробити такі класи, як Cardiologist,Dashboard, Visit. Кожен клас має властивості, що описують його стан, і методи для маніпулювання цим станом.

2. Успадкування
Спадкування дозволяє новим класам успадковувати властивості та методи існуючих класів. Наприклад, класи Cardiologist і Dentist можуть успадковувати від базового класу Doctor. Такий підхід зменшує дублювання коду та спрощує управління кодом.

3. Поліморфізм
Поліморфізм в ООП дозволяє об'єктам приймати різні форми. Наприклад, можна визначити інтерфейс Dashboard з методом manage(); різні класи можуть реалізувати цей інтерфейс, щоб забезпечити специфічну реалізацію методу.

4. Інкапсуляція
Інкапсуляція допомагає обмежити доступ до даних класу, контролюючи, як зовнішній код взаємодіє з внутрішнім станом об'єкта. Це робить дані більш захищеними і запобігає виникненню помилок при використанні класу. Наприклад, до закритих полів класу AdminPanel можна отримати доступ лише через загальнодоступні методи (наприклад, getAdminPanel() та setAdminPanel()).

5. Модульність
Завдяки ООП програмне забезпечення можна розділити на модулі, які можна розробляти, тестувати, оптимізувати та оновлювати незалежно один від одного. Наприклад, можна розробити окремі модулі для управління пацієнтами, лікарями, прийомами та розкладом, що спрощує керування проектами та підвищує якість кінцевого продукту.

Висновок до розділу 1
Об'єктно-орієнтоване програмування (ООП) пропонує значні переваги для розробки програмного забезпечення для управління обліком прийомів, оскільки дозволяє інтуїтивно зрозуміле моделювання реальних об'єктів і процесів. Завдяки інкапсуляції, успадкуванню та поліморфізму ООП полегшує створення модульного, масштабованого коду, який легко розширювати та підтримувати; ООП підвищує ефективність розробки завдяки повторному використанню коду та зменшує ймовірність помилок завдяки жорсткому контролю доступу до даних та методів. Крім того, ООП полегшує співпрацю між розробниками, оскільки структурований код у вигляді класів та об'єктів легше інтерпретувати та аналізувати. Такий підхід дозволяє командам ефективніше керувати великими проектами, розподіляючи роботу та мінімізуючи залежності між різними частинами системи. Використання принципів ООП також забезпечує вищу безпеку розробки, обмежуючи можливості використання інтерфейсів та даних. Таким чином, ООП не тільки спрощує процес розробки, але й дозволяє програмному забезпеченню ефективно адаптуватися та відповідати вимогам динамічної галузі охорони здоров'я.

Вибір середовища та інструментів розробки
Великий обсяг даних та доступ до мережі дає змогу розробляти програмні додатки для запису навчальних програм, використовуючи JavaScript (JS) як базову технологію. JavaScript - це дуже гнучка та потужна мова програмування. Деякі з основних аспектів і технологій, які можуть бути використані під час розроблення таких додатків, такі як:
1. Node.js
Node.js - це серверне середовище виконання для JavaScript, яке дає змогу розробляти бекенди додатків на JS. Node.js може бути використаний для створення швидких і Масштабовані бекенди можуть бути створені.

2. Express.js як серверний фреймворк
Express.js - це легкий фреймворк для Node.js, який спрощує розробку серверних додатків завдяки проміжному ПЗ та простій маршрутизації. Він допомагає створювати RESTful API, які взаємодіють із клієнтською частиною програми.
 3. React.js на стороні клієнта
React.js - одна з найпопулярніших бібліотек для створення користувацьких інтерфейсів. Вона дає змогу створювати динамічні та інтерактивні веб-інтерфейси й ідеально підходить для керованих додатків, де користувачам потрібен зручний та інтуїтивно зрозумілий інтерфейс.
4. Бази даних і ORM
Дані можуть зберігатися в різних базах даних, включно з MongoDB, PostgreSQL і MySQL, а ORM-бібліотеки, як Mongoose (для MongoDB) і Sequelize (для баз даних SQL), можна використовувати для ефективного управління даними та забезпечення серверної та взаємодії між сервером і базою даних.

5. Безпека та автентифікація
За допомогою таких пакетів, як Passport.js, у застосунку можна реалізувати аутентифікацію та авторизацію. Це важливо для забезпечення безпеки даних і контролю доступу до різних частин системи.
6. Єдина кодова база
Використання JavaScript для всіх компонентів застосунку (клієнт, сервер і доступ до даних) дає змогу розробникам мати єдину кодову базу, що спрощує розробку та супровід програмних продуктів. Загалом JavaScript пропонує сучасний та ефективний підхід до розроблення програмного забезпечення, значно підвищуючи продуктивність розроблення та забезпечуючи високу якість кінцевого продукту для управління дисципліною або навчальним планом навчального закладу.

Модальне вікно "Створити візит"
У модальному вікні повинні бути:

Випадаючий список (select) з вибором лікаря. Залежно від обраного лікаря, під цим списком будуть з'являтися поля, які потрібно дозаповнити для візиту до цього лікаря.
У списку має бути три опції - Кардіолог, Стоматолог, Терапевт.
Після вибору лікаря зі списку, під ним повинні з'явитися поля для запису до цього лікаря. Декілька полів є однаковими для всіх трьох докторів: мета візиту, короткий опис візиту, дропдаун - терміновість (звичайна, пріоритетна, невідкладна), ПІБ
Також кожен з лікарів має свої унікальні поля для заповнення. 
Якщо вибрано опцію Кардіолог, додатково з'являються такі поля для введення інформації: звичайний тиск, Індекс маси тіла, перенесені захворювання серцево-судинної системи, вік
Якщо вибрано опцію Стоматолог, додатково необхідно заповнити: дата останнього відвідування
Якщо вибрано опцію Терапевт, додатково необхідно заповнити: вік
Кнопка Створити. При натисканні на кнопку надсилається AJAX запит на відповідну адресу, і якщо у відповіді надійшла інформація про новостворену картку - створюється картка у Дошці візитів на сторінці, модальне вікно закривається.
Кнопка Закрити - закриває модальне вікно без збереження інформації та створення картки. По кліку на область поза модальним вікном - модальне вікно також закривається.
Усі поля вводу, незалежно від обраної опції, крім поля для додаткових коментарів, є обов'язковими для введення даних.

3.2 Проблеми, які виникли при розробці
Приватизація даних та роутинг
Медична інформація є чутливою, тому її захист має вирішальне значення. Через те, що хостинг такого сервісу вимагає певних ресурсів, я не бачив сенсу налаштувати відповідний рівень захисту даних. Тому було вирішено зробити сторінку автентифікації. 
Бази даних мають постійно поповнюватися через запити на сервер. Конкретно цей застосунок під'єднаний  до серверної частини через ajax запити, але на мою думку можна було використати сервіс MongoDB з персональними налаштуваннями Шляхів (React Routes), динамічним хостингом (MongoDb Atlas dynamic infrastructure) та іншими функціями (Див. малюнок 9)

Сертифікація
Додатки для обліку пацієнтів повинні інтегруватися з різними медичними системами для обміну даними, що вимагає уніфікованих стандартів та надійних протоколів передачі даних, але інтеграція такого протоколу потребує сертифікації, що є неможливим для некомерційного застосунку. 
Debugging
Підтримка сервісу повинна мати чітку структуру та логіку. Розробка стратегій для відновлення діяльності після збоїв та регулярне резервне копіювання критично важливих даних забезпечує стійкість системи до непередбачених подій. Так як я не маю відповідних навичок для створення стабільної системи, я можу забезпечити підтримку коду на поверхневому рівні.

Застосування мови програмування JavaScript допомогло створити інтерактивний користувацький інтерфейс. Також, для оптимізації взаємодії з базою даних та забезпечення безпечного зберігання інформації були використані різні аспекти серверної логіки і роботи з API, що забезпечило надійність та високу продуктивність системи.
